springboot autoconfigure

大体流程

spring-boot-autocofigure自动配置启动springsecurity，实例化并注册自带配置类SecurityAutoConfiguration、创建一个UserDetailsS​​ervice和SecurityFilterAutoConfiguration。

SecurityAutoConfiguration主要作用是配置默认认证发布器DefaultAuthenticationEventPublisher，并映入SpringBootWebSecurityConfiguration、WebSecurityEnablerConfiguration和SecurityDataConfiguration这三个配置类，SpringBootWebSecurityConfiguration会根据是否以及有配置WebSecurityConfigurerAdapter的Bean来决定是否实例化自身并提供和注册Bean默认WebSecurityConfigurerAdapter的Bean;WebSecurityEnablerConfiguration则添加@EnableWebSecurity的注解来启用SpringSecurity的webSecurity

主要内容

springboot的自动配置功能都集成在spring-boot-autocofigure.jar包下

在spring-boot-autocofigure.jar包中的META-INF/spring.factories，有关于spring-security的部分

其中有关于spring-security-web部分的有SecurityAutoConfiguration，UserDetailsServiceAutoConfiguration，SecurityFilterAutoConfiguration，springboot启动时会进行实例化并注入Spring IOC容器

Spring Boot Auto Configuration
Spring Boot 自动地:

  1. 启用S​​pring Security的默认配置

    启用S​​pring Security的默认配置，该配置将Servlet过滤器创建为名为springSecurityFilterChain的bean。此bean负责应用程序内的所有安全性（保护应用程序URL，验证提交的用户名和密码，重定向到登录表单等）。

spring-boot-autoconfigure模块中的org.springframework.boot.autoconfigure.security.servlet包中有SecurityAutoConfiguration、WebSecurityEnablerConfiguration和SpringBootWebSecurityConfiguration三个配置类。现在从启动时实例化并注入IOC容器的SecurityAutoConfiguration开始分析。

注解EnableGlobalAuthentication.class

EnableGlobalAuthentication主要有两个作用

  • 被@EnableGlobalAuthentication所注解的类，可以配置一个AuthenticationManagerBuilder的全局实体
  • 被@EnableGlobalAuthentication所注解的注解类，该注解所修饰的类可以配置一个AuthenticationManagerBuilder的全局实体，@EnableGlobalAuthentication注解过的注解类有EnableWebSecurity、EnableWebMvcSecurity和EnableGlobalMethodSecurity，也就是说EnableWebSecurity、EnableWebMvcSecurity和EnableGlobalMethodSecurity注解过的类可以配置一个AuthenticationManagerBuilder的全局实体
